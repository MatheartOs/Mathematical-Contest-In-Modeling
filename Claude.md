## 2026年数维杯B题赛题解析

### 一、题目整体分析

#### 1.1 赛题背景与宏观定位

本题面向**智能办公场景下的多源异构文件识别与治理优化问题**。题目所给数据既包含历史真实文件，又包含后续流入的半结构化记录和匿名原始文件，文件格式、文本完整程度、结构层次和业务属性均不一致。

本题是异构数据统、表征、历史主题体系构建、新数据迁移归属、人工复核与资源调度优化的综合建模问题。

**三问递进关系：**
- 问题一：从数据集1中挖掘稳定的类别结构和主题规律
- 问题二：将分类体系迁移到数据集2和数据集3，对归属结果的合理性、可解释性和迁移适用性进行量化评价
- 问题三：结合分类置信度和业务风险，在资源受限条件下判断哪些文件需要人工复核以及复核优先级

#### 1.2 核心矛盾分析

1. **多源异构数据与统一特征空间之间的矛盾**：历史文件可能来自Word、PDF、Excel、图片或文本等不同载体，必须先进行统一解析、清洗和标准化编码。

2. **历史完整文件与后续不完整数据之间的矛盾**：数据集2、数据集3中存在标题、片段、来源信息等不完整内容，分类模型需要具备迁移能力。

3. **自动归档效率与人工复核安全性之间的矛盾**：需要设计自动分类、风险评价、人工复核的人机协同机制。

#### 1.3 建模逻辑架构

采用分层闭环建模思路：
1. **数据统一层**：对多源文件进行格式解析、文本抽取、清洗规范化和元数据保留
2. **分类建类层**：基于文本-单词异构图构建图神经网络分类模型，利用BERT表征和GCN聚合能力
3. **迁移评价层**：对流入数据进行类别归属判断，设计归属合理性、模型可解释性、迁移适用性等指标
4. **治理优化层**：根据紧急程度、错分风险和复核必要性构建文件复核优先级得分，建立多目标混合整数规划模型

---

### 二、数据预处理与特征工程

#### 2.1 版面分析与文本提取

- **图像和扫描版PDF**：采用OCR识别与版面分析相结合的方法，识别标题、正文、表格、印章和签名等区域
- **原生PDF和Word文件**：用PDF解析工具或python-docx等方式提取正文文本，保留标题层级、字号、加粗、表格结构和页码等信息
- **Excel或半结构化记录**：用字段名、字段值的序列化方式展开，例如"[标题]：……；[来源]：……；[正文片段]：……"
- **纯文本文件**：统一转换为UTF-8编码，处理异常换行、不可见字符、重复标点和无意义乱码

#### 2.2 文本清洗与规范化

- 利用正则表达式删除连续特殊符号、页眉页脚、水印残留和重复空白
- 结合办公场景词典，对业务关键词、组织名称、金额、日期、编号和文件来源等信息进行规范化
- 对于需要输入BERT或SBERT的文本，不宜过度分词或删除句法结构

### 三、各小题详细分析

#### 3.1 问题一：历史多源异构文件分类与主题归纳

**核心任务**：利用数据集1中的历史真实文件，挖掘关键信息并进行分类，同时归纳各类别对应的主题。

**建模核心步骤**：

1. **构建标准文本特征空间**：对不同格式文件进行解析后，提取标题、正文、关键词、来源、时间、表格字段和版面结构等信息。对于长文本，采用分段编码后池化方式形成文档向量；对于短文本，可更多依赖标题、来源和关键词信息补足语义。

2. **构建文本-单词异构图**：设无向异构图 G = (V, E), V = V_D ∪ V_W，其中V_D为文档节点集合，V_W为单词节点集合。邻接矩阵A定义为：
   - PMI(i,j), i,j ∈ V_W, PMI(i,j) > 0
   - TFIDF, i ∈ V_D, j ∈ V_W
   - 1, i = j
   - 0, otherwise

   PMI计算公式：
   ```
   PMI(i,j) = log(p(i,j)·N / (n(i)·n(j)))
   ```
   其中n(i,j)表示词i和词j在滑动窗口中共同出现的次数，n(i)表示包含词i的窗口数，N表示滑动窗口总数。

3. **初始化节点特征**：
   - 文档节点：使用预训练BERT模型提取[CLS]向量作为初始特征
   - 单词节点：使用词嵌入向量初始化，或采用零向量后通过图结构逐层更新

4. **利用GCN聚合语义与共现信息**：
   - 度矩阵 D_ii = Σ_j A_ij
   - 归一化邻接矩阵：A' = D^(-1/2) · A · D^(-1/2)
   - 图卷积更新公式：H^(l+1) = σ(A' · H^(l) · W^(l)), H^(0) = X

5. **基于BERTopic或c-TF-IDF归纳类别主题**：
   - c-TF-IDF公式：W_{t,c} = tf_{t,c} · log(1 + A / tf_t)
   - 其中tf_{t,c}表示词t在主题c中的频率，tf_t表示词t在所有主题中的总频率，A表示每个主题的平均词汇量

#### 3.2 问题二：新流入数据归属判断与综合评价

**核心任务**：将问题一建立的分类体系迁移到数据集2和数据集3，对半结构化记录和匿名原始文件进行归类，并设计评价指标。

**建模核心步骤**：

1. **对新数据进行一致化预处理**：数据集2为后续流入的半结构化记录，数据集3为匿名真实文件

2. **进行类别迁移归属判断**：将新数据输入分类模型，得到每个样本属于K个类别的概率分布

3. **构建归属合理性得分ARS**：
   ```
   ARS = α·p_max + (1-α)·(p_max - p_second) / p_max, 0 < α < 1
   ```
   ARS越接近1，说明该样本归属越清晰

4. **构建模型可解释性指数MII**：
   ```
   MII = 1 - (Σ a_i · log(a_i)) / log(L)
   ```
   其中L为有效Token数，A为注意力权重。当MII较大时，说明模型的注意力更加聚焦

5. **构建迁移适用性指标TAI**：
   ```
   TAI = exp(-γ · MMD²(X_s, X_t)), γ > 0
   ```
   TAI越接近1，说明新数据与历史数据分布越接近

6. **建立边界样本与异常样本识别逻辑**：
   - 状态A（清晰样本）：p_max ≥ θ_conf, ARS ≥ 0.8，可直接自动归档
   - 状态B（多类别重叠样本）：p_max < θ_conf, p_max - p_second < ε
   - 状态C（未知或分布外样本）：最大概率较低，且预测概率分布的信息熵接近最大熵

#### 3.3 问题三：文件人工复核优先级与资源约束调度优化

**核心任务**：结合问题二的归类结果，从紧急程度、错分风险和复核必要性三个方面对待处理文件进行等级划分，在三种资源约束场景下判断哪些文件需要人工复核及优先顺序。

**建模核心步骤**：

1. **定义三类核心评价指标**（均归一化到[0,1]）：
   - 紧急程度U_i：由时效要求、截止时间、业务流程节点等信息转化
   - 错分风险R_i：可采用最大概率补集或归一化熵
   - 复核必要性N_i：根据业务规则赋值（主题归属不明确、包含合同/公章/签名、涉及资金分配等）

2. **采用AHP-熵权法确定综合权重**：
   - AHP：体现主观业务偏好
   - 熵权法：反映数据本身差异程度
   - 综合权重：w_j = (w_j^AHP · w_j^entropy) / Σ(w_k^AHP · w_k^entropy)

3. **构建文件复核优先级得分**：
   ```
   Score_i = (w_U · U_i + w_R · R_i + w_N · N_i) · (1 + μ · I特殊文件), μ > 0
   ```

4. **划分高、中、低复核等级**：采用分位数法或K-Means聚类法

5. **建立人工复核多目标优化模型**：
   - 时效性目标：Z_1 = max Σ t_ij · x_ij
   - 人工复核成本：Z_2 = Σ Σ c_ij · t_ij · x_ij
   - 未复核错分惩罚：Z_3 = Σ (1 - y_i) · RC_penalty
   - 综合目标函数：min F = α_1·Z_1* + α_2·Z_2* + α_3·Z_3*

6. **约束条件**：
   - 每个文件最多分配给一名复核人员
   - 高等级文件必须复核
   - 时间受限：Σ t_ij · x_ij ≤ T_max
   - 预算受限：Σ Σ t_ij · x_ij ≤ B_max
   - 专家人力受限：Σ Σ t_ij · x_ij ≤ L_max

7. **采用遗传算法求解**：将长度为N的整数序列作为染色体，基因g_i表示文件i的处理方式（0表示不进入人工复核，j表示分配给人员j）
