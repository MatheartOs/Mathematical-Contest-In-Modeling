# Problem 1 论文描述：历史多源异构文件分类与主题归纳

> 本文档用于支撑数维杯 B 题论文中“问题一：历史多源异构文件分类与主题归纳”部分的撰写。内容对应当前实现 `src/mcm_b/problem1_innovative.py` 与运行结果 `outputs/b_problem/problem1_innovative/`。论文手可直接据此改写为论文正文、公式说明、算法流程和结果分析。

## 1. 问题一目标

问题一要求利用数据集 1 中的历史真实文件，完成三项任务：

1. 对 Word、PDF、Excel、图片、TXT 等多源异构文件进行统一特征挖掘。
2. 基于提取结果对历史文件进行自动分类。
3. 归纳各类别对应的主题名称、关键词、代表文件和业务解释。

本项目采用“统一文件画像 + 文本-单词异构图 + PMI 词图传播 + 结构业务特征融合 + 业务锚点增强 + c-TF-IDF 主题归纳”的方法实现问题一。该方法借鉴 `docs/解析.md` 中关于 BERT、GCN、BERTopic、c-TF-IDF 的建模建议，但考虑当前竞赛环境没有稳定的深度学习依赖，最终实现为一个可复现、可解释、计算成本较低的轻量创新版本。

核心思想是：先将每个历史文件转化为统一文本和结构业务特征，再构建文档-词、词-词两类关系。文档-词关系由 TF-IDF 表示，词-词关系由滑动窗口 PMI 表示。随后通过两跳图传播，将词语共现结构融入文档向量，相当于对 GCN 聚合思想进行轻量近似。最后融合文件结构、业务属性和主题锚点特征，完成聚类与主题归纳。

## 3. 统一建模文本构造

每个文件清洗后已经具有标题、正文、OCR 文本、业务关键词等字段。问题一将这些字段合并为建模文本：

```text
model_text_i = title_i + clean_text_i + business_keywords_i
```

为了减少 OCR 数字表格、英文编码、Excel 模板字段对聚类的干扰，进一步进行中文语义过滤：

1. 删除英文、数字、编号和小数形式。
2. 只保留中文字符。
3. 删除固定模板词，如“工作表名称”“表头字段”“主要关键词”“样例内容”“图片名称”“图片编号”“下载”“关键词”。

记第 `i` 个文件的建模文本为：

```text
d_i = Clean(title_i, clean_text_i, business_keywords_i)
```

这一处理保证主题模型主要根据业务语义、中文关键词和文件内容共现关系分类，而不是被表格编号、小数、URL 或模板字段带偏。

## 4. 文档-词 TF-IDF 表示

由于中文文件未进行依赖外部分词库的分词，本模型采用字符级 n-gram 表示。具体使用 2 至 4 字符片段：

```text
g_t ∈ G = {所有 2-gram, 3-gram, 4-gram 中文片段}
```

设共有 `N` 个历史文档，筛选后词项数为 `M`。程序参数为：

| 参数 | 当前值 | 含义 |
| --- | ---: | --- |
| `ngram_range` | `(2, 4)` | 使用 2 至 4 字符中文片段 |
| `min_df` | `2` | 至少出现在 2 个文档中 |
| `max_df` | `0.88` | 出现在 88% 以上文档中的泛化词过滤 |
| `max_features` | `2500` | 最多保留 2500 个词项 |
| `sublinear_tf` | `True` | 对词频使用对数缩放 |

得到文档-词 TF-IDF 矩阵：

```text
X ∈ R^{N × M}
```

其中：

```text
X_{i,t} = TFIDF(d_i, g_t)
```

TF-IDF 可写为：

```text
tfidf_{i,t} = tf_{i,t} * log((N + 1) / (df_t + 1))
```

其中 `tf_{i,t}` 表示词项 `g_t` 在文档 `d_i` 中的频率，`df_t` 表示包含该词项的文档数。

该矩阵同时可以理解为异构图中的文档-词边权：

```text
A_{d_i, w_t} = TFIDF(d_i, g_t)
```

## 5. 词-词 PMI 共现图

为刻画词语之间的全局共现关系，模型进一步构建词-词图。程序将每个文档转化为词项序列，并使用滑动窗口统计词项共现。

当前参数：

```text
window_size = 18
```

设：

| 符号 | 含义 |
| --- | --- |
| `n_W(t)` | 包含词项 `t` 的滑动窗口数 |
| `n_W(t,s)` | 同时包含词项 `t` 和 `s` 的滑动窗口数 |
| `N_W` | 全部滑动窗口数 |

则词项 `t` 与 `s` 的 PMI 定义为：

```text
PMI(t,s) = log( p(t,s) / (p(t)p(s)) )
```

代入窗口统计量：

```text
PMI(t,s) = log( n_W(t,s) * N_W / (n_W(t) * n_W(s)) )
```

只有当 `PMI(t,s) > 0` 时才保留边：

```text
W_{t,s} =
  PMI(t,s),  if PMI(t,s) > 0
  0,         otherwise
```

为了避免极端高 PMI 边影响稳定性，代码中将 PMI 上限截断为 8：

```text
W_{t,s} = min(PMI(t,s), 8)
```

随后对词图按行归一化：

```text
\hat{W} = D_w^{-1} W
```

其中 `D_w` 为词图的行度矩阵。

本次运行得到：

```text
word_graph_windows = 404135
word_graph_edges = 366841
term_count = 2500
```

这说明模型从历史文件中学习到了较丰富的词项共现结构。

## 6. 文本-单词异构图表示

按照解析文档建议，可构建文本-单词异构图：

```text
G = (V, E),  V = V_d ∪ V_w
```

其中：

| 节点集合 | 含义 |
| --- | --- |
| `V_d` | 文档节点集合 |
| `V_w` | 词项节点集合 |

边包括两类：

1. 文档-词边：由 TF-IDF 表示。
2. 词-词边：由正 PMI 表示。

完整邻接关系可写为：

```text
A_{ij} =
  PMI(i,j),      i,j ∈ V_w 且 PMI(i,j)>0
  TFIDF(i,j),    i ∈ V_d, j ∈ V_w
  1,             i = j
  0,             otherwise
```

在论文中可说明：由于历史文件没有人工标签，且当前环境不依赖深度学习框架，本文未直接训练完整 GCN，而是通过词图传播近似实现 GCN 的语义聚合效果。这一设计保留了异构图思想，同时保证模型可复现。

## 7. 两跳图传播近似 GCN 聚合

GCN 的核心是利用邻接矩阵传播邻居信息：

```text
H^{(l+1)} = σ(\tilde{A} H^{(l)} W^{(l)})
```

对于本题，文档节点和词项节点的主要关系已经由 TF-IDF 与 PMI 表示。因此可将“文档-词-词”路径转化为矩阵传播：

```text
X^{(0)} = X
X^{(1)} = X \hat{W}
X^{(2)} = X^{(1)} \hat{W}
```

其中：

| 矩阵 | 含义 |
| --- | --- |
| `X^{(0)}` | 原始文档-词 TF-IDF 表示 |
| `X^{(1)}` | 一跳词图传播，融入直接共现词 |
| `X^{(2)}` | 二跳词图传播，融入间接共现词 |

最终图增强文本向量定义为：

```text
X_graph = X^{(0)} + 0.60 X^{(1)} + 0.30 X^{(2)}
```

即：

```text
X_graph = X + 0.60 X\hat{W} + 0.30 X\hat{W}^2
```

该公式的含义是：

1. 保留原文档中的直接关键词。
2. 融入与关键词强共现的相关词。
3. 少量融入二阶相关词，增强主题语义连贯性。

这种方法可理解为一个无监督、无需训练参数的轻量 GCN 近似，适用于没有人工标签的大规模办公文件主题发现任务。

## 8. 结构特征与业务特征融合

仅依靠文本可能无法充分表达多源异构文件的形态差异。例如 Excel 表、扫描图片、通知文件、合同文件在结构上明显不同。因此模型将结构特征和业务特征拼接进最终表示。

| --- | --- |
| `has_notice` | 是否包含通知/公告线索 |
| `has_meeting` | 是否包含会议线索 |

所有结构和业务特征先进行标准化：

```text
z_{i,j} = (x_{i,j} - μ_j) / σ_j
```

记标准化后的结构业务矩阵为：

```text
S ∈ R^{N × P}
```

### 8.3 业务锚点增强特征

全量重跑检查发现，仅依靠无监督聚类时，主题容易被“地区、单位、服务、项目”等办公泛词牵引。为增强主题可解释性，最终模型增加业务锚点特征。设共有 `G` 个业务锚点组，第 `g` 组锚词集合为 `A_g`，则第 `i` 个文档在该锚点组上的原始得分为：

```text
a_{i,g} = log(1 + sum_{t in A_g} count(t, d_i))
```

归一化后得到业务锚点矩阵：

```text
C ∈ R^{N × G}
```

当前锚点组覆盖资金财政、医药审批、生态环境、教育教学、养老服务、制造业、居民收入、社会民生、城市指标和项目案件等方向。锚点特征权重为：

```text
anchor_embedding_weight = 3.00
```

则最终融合向量为：

```text
F = [X_graph, S, C]
```

并做行归一化：

```text
\tilde{F}_i = F_i / ||F_i||_2
```

## 9. 降维与聚类分类

由于融合后的矩阵维度较高，先使用 TruncatedSVD 降维：

当前降维维度：

```text

```


## 10. c-TF-IDF 主题词归纳

分类完成后，需要解释每个类别的主题。程序采用 c-TF-IDF 方法提取每类代表词。

设第 `c` 个类别中所有文档合并为一个“类文档”：

```text
D_c = concat({d_i | y_i = c})
```

对每个词项 `t`，计算其在类别 `c` 中的词频：

```text
tf_{t,c}
```

设 `df_t^c` 为词项 `t` 出现过的类别数或全局类频率，`\bar{L}` 为平均类别词项长度，则 c-TF-IDF 权重定义为：

```text
W_{t,c} = tf_{t,c} * log(1 + \bar{L} / df_t^c)
```

代码中按照上述思想，对每类词项计数并用类别频率进行惩罚。若某词在某一类高频出现、但在其他类很少出现，则其权重较高，可作为该主题的代表词。

每个类别最终输出：

1. `ctfidf_terms`：该类 c-TF-IDF 权重最高的主题词。
2. `topic_name`：依据主题词、业务画像和代表文件生成的主题名称。
3. `topic_explanation`：该类为何形成、有哪些结构或业务特征。

## 11. 代表文件选择

## 13. 聚类评价指标

为了衡量聚类结构，程序输出三个无监督评价指标。

### 13.1 轮廓系数 Silhouette

对样本 `i`，设：

| 符号 | 含义 |
| --- | --- |
| `a_i` | 样本到同簇样本的平均距离 |
| `b_i` | 样本到最近其他簇样本的平均距离 |

轮廓系数：

```text
s_i = (b_i - a_i) / max(a_i, b_i)
```

总体轮廓系数为所有样本的平均值：

```text
S = mean(s_i)
```

值越大表示类内越紧密、类间越分离。

### 13.2 Calinski-Harabasz 指数

该指标衡量类间离散度与类内离散度的比值：

```text
CH = (B_k / (K-1)) / (W_k / (N-K))
```

其中 `B_k` 为类间离散度，`W_k` 为类内离散度。值越大通常表示聚类效果越好。

### 13.3 Davies-Bouldin 指数

该指标衡量簇内散度与簇间距离的相对关系：

```text
DB = (1/K) * sum_i max_{j≠i} ((S_i + S_j) / M_{ij})
```

其中 `S_i` 为第 `i` 簇内部散度，`M_{ij}` 为第 `i`、`j` 簇中心距离。值越小越好。

本次运行结果：

```text
silhouette = 0.035421
calinski_harabasz = 39.333307
davies_bouldin = 3.439216
```

需要注意：历史数据中存在大量统计图片、表格型 OCR 文本和跨主题办公材料，类别边界天然重叠，因此轮廓系数偏低并不意味着模型不可用。论文中建议解释为“办公文件主题具有交叉性，模型更强调可解释归纳与后续迁移归档，而不是严格球形簇分离”。

## 16. 可直接写入论文的方法描述

论文中可采用如下文字：

```text
针对数据集 1 中 Word、PDF、Excel、图片和 TXT 等多源异构历史文件，本文首先基于前序清洗结果构建统一文件画像，保留标题、正文、OCR 文本、表格语义、结构特征和业务属性。为增强分类模型对异构文本的表达能力，本文构建文本-单词异构图：以历史文档和中文 n-gram 词项作为节点，以 TF-IDF 表示文档-词边，以滑动窗口正 PMI 表示词-词边。

在图表示阶段，本文采用两跳词图传播近似 GCN 的邻域聚合过程，将与核心词高共现的一阶和二阶相关词融入文档向量，得到图增强文本表示。随后将页数、段落数、表格数、图片数、OCR 置信度、清洗质量以及资金、项目、合同、会议、截止、紧急等业务标记进行标准化，并加入业务锚点增强特征，与图增强文本向量拼接，形成最终文件表示。

在分类阶段，本文使用 TruncatedSVD 对高维融合表示降维，并采用 KMeans 对历史文件进行聚类。为解释每一类主题，本文将同一类别下所有文档合并为类文档，使用 c-TF-IDF 提取该类相对于其他类别更具代表性的关键词，并结合业务画像和代表文件自动生成主题名称。最终得到 10 类历史主题体系。
```

## 17. 可直接写入论文的数学流程

可将问题一模型概括为以下数学流程：

```text
RawFile_i -> CleanedDocument_i = {T_i, L_i, S_i, B_i, Q_i}
```

其中 `T_i` 是文本，`L_i` 是版面块信息，`S_i` 是结构特征，`B_i` 是业务属性，`Q_i` 是清洗质量。

构造文档-词矩阵：

```text
X_{i,t} = TFIDF(d_i, g_t)
```

构造词-词 PMI 图：

```text
W_{t,s} = max(0, log(n_W(t,s)N_W / (n_W(t)n_W(s))))
```

行归一化：

```text
\hat{W} = D_w^{-1}W
```

两跳图传播：

```text
X_graph = X + β_1 X\hat{W} + β_2 X\hat{W}^2
```

当前取：

```text
β_1 = 0.60, β_2 = 0.30
```

拼接结构业务特征：

```text
F = Normalize([X_graph, S, C])
```

降维：

```text
E = SVD(F)
```

聚类：

```text
y_i = KMeans(E_i),  y_i ∈ {1,2,...,K}
```

主题词：

```text
W_{t,c} = tf_{t,c} log(1 + \bar{L}/df_t^c)
```

其中 `K=10`。

## 18. 模型亮点

该问题一实现有以下可写入论文的创新点：

1. 不只使用纯文本，而是融合文本、结构、业务、质量和业务锚点五类文件画像信息。
2. 采用文档-词 TF-IDF 与词-词 PMI 构成异构图，贴合赛题解析中建议的文本-单词图结构。
3. 通过两跳词图传播近似 GCN 聚合，无需人工标签和深度学习环境即可利用词语共现结构。
4. 对 OCR 统计表噪声进行中文语义过滤，降低数字、小数和模板字段对主题分类的干扰。
5. 增加业务锚点增强，降低办公泛词对聚类中心的牵引，使主题体系更稳定、更接近业务语义。
6. 使用 c-TF-IDF 进行类别主题词归纳，输出主题名称、代表文件和业务画像，便于论文解释。
7. 输出文件级分类结果和类别级汇总结果，可直接衔接问题二迁移分类与问题三复核风险评估。

## 19. 局限与论文表述建议

当前模型仍有以下局限：

1. 因当前环境没有稳定 BERT / torch 依赖，未直接训练深度 GCN，而是采用图传播近似。
2. 数据集 1 中大量文件是统计图片或 OCR 表格，部分主题边界天然交叉，导致轮廓系数较低。
3. `资金财政统计类` 与 `医药项目审批类` 样本数相对较多，若论文篇幅允许，可在讨论中说明这两类可进一步做二级主题拆分。
4. 自动主题命名需要结合代表文件进行人工润色，论文中建议称为“模型建议主题名”或“归纳主题名”。

论文中不宜只强调聚类指标，而应强调本题是办公文件治理问题，模型目标是建立可解释、可迁移、可用于复核调度的主题体系。因此，类别解释、代表文件和业务画像比单一聚类指标更重要。
