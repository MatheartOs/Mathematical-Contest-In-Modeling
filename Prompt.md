## 一、题目概述

### 1.1 问题背景

## 三、问题一：历史多源异构文件分类与主题归纳

### 核心任务
对数据集1中的多源异构文件进行特征挖掘，提取反映文件内容及属性的关键信息，基于提取结果对文件分类，并归纳各类别对应的主题。

### 推荐建模流程

#### 步骤1：数据统一层（格式解析与文本提取）
- **图像/扫描版PDF**：OCR识别 + 版面分析，识别标题、正文、表格、印章等区域
- **原生PDF/Word**：PDF解析工具/python-docx提取正文，保留标题层级、字号、加粗等
- **Excel/半结构化记录**：字段名-字段值序列化，如"[标题]：……；[来源]：……；[正文片段]：……"
- **纯文本文件**：统一UTF-8编码，处理异常换行、不可见字符

#### 步骤2：文本清洗与规范化
- 正则表达式删除连续特殊符号、页眉页脚、水印残留
- 办公场景词典规范化业务关键词、组织名称、金额、日期、编号等

#### 步骤3：构建文本-单词异构图
```
G = (V, E), V = V_doc ∪ V_word
```
- 邻接矩阵A定义：
  - PMI(i,j), i,j ∈ V_word（单词共现连接）
  - TFIDF(i,j), i ∈ V_doc, j ∈ V_word（文档-单词连接）
  - 1, i = j（自连接）

#### 步骤4：节点特征初始化
- **文档节点**：BERT提取[CLS]向量作为初始特征
- **单词节点**：词嵌入向量初始化

#### 步骤5：GCN图卷积聚合
```python
H^(l+1) = σ(Â H^(l) W^(l))
```
经过两层GCN后提取文档节点表示，通过Softmax输出类别概率

#### 步骤6：主题归纳（BERTopic或c-TF-IDF）
- 使用SBERT得到类别内文档向量
- UMAP降维 + HDBSCAN聚类识别细分主题
- c-TF-IDF提取主题标签关键词

## 四、问题二：新流入数据归属判断与综合评价

### 核心任务
根据问题一建立的分类体系，对数据集2和数据集3中的数据进行归类，设计评价指标对分类效果、归属合理性、可解释性和迁移适用性进行综合量化评价。

### 推荐建模流程

#### 步骤1：新数据一致化预处理
- 数据集2：重点保留标题、正文片段、来源和时间字段
- 数据集3：补充OCR、格式解析和版面结构特征

#### 步骤2：类别迁移归属判断
```python
P(i) = [p_1(i), p_2(i), ..., p_K(i)], Σp = 1
y_pred = argmax p_k
```

#### 步骤3：归属合理性得分 ARS（Assignment Rationality Score）
```python
ARS = α * p_max + (1-α) * (p_max - p_second) / p_max
```
- p_max：绝对置信度
- (p_max - p_second)/p_max：分类边界安全裕度

#### 步骤4：模型可解释性指数 MII（Model Interpretability Index）
```python
MII = 1 - (-ΣL a_i log a_i) / log L
```
- 基于注意力权重分布：注意力越聚焦，MII越高

#### 步骤5：迁移适用性指标 TAI（Transferability Applicability Index）
```python
TAI = exp(-γ * MMD(X_source, X_target))
```
- 使用最大均值差异MMD衡量源域和目标域表征一致性

#### 步骤6：样本状态分类
| 状态 | 条件 | 处理方式 |
|------|------|----------|
| A：清晰样本 | p_max ≥ θ_conf, ARS ≥ 0.8 | 直接自动归档 |
| B：多类别重叠样本 | p_max < θ_conf, p_max - p_second < ε | 进入人工复核池 |
| C：未知/分布外样本 | 最大概率低，熵接近最大熵 | 交由专家研判 |

### 输出建议
- 每个样本：预测类别、最大置信度、次高类别、ARS、MII、TAI、状态标记
- 单独列出状态B和状态C样本及其进入人工复核的原因

## 五、问题三：文件人工复核优先级与资源约束调度优化

### 核心任务
从紧急程度、错分风险和复核必要性三个方面对待处理文件进行高、中、低等级划分，判断各文件是否需要人工复核及复核优先顺序，在三种资源约束场景下比较处理结果差异。

### 推荐建模流程

#### 步骤1：定义三类核心指标（归一化到[0,1]）
- **紧急程度U_i**：由时效要求、截止时间、业务流程节点等信息转化
- **错分风险R_i**：
  ```python
  R_i = 1 - p_max  # 或使用归一化熵
  ```
- **复核必要性N_i**：根据业务规则赋值（主题归属不明确、合同/公章/签名、资金分配、来源异常、文件完整性不足时提高）

#### 步骤2：AHP-熵权法确定综合权重
- **AHP**：体现主观业务偏好
- **熵权法**：反映数据本身差异程度
- **综合权重**：
  ```python
  w_j = (w_主观_j * w_客观_j) / Σ(w_主观 * w_客观)
  ```

#### 步骤3：构建复核优先级得分
```python
Score_i = (w1*U_i + w2*R_i + w3*N_i) * (1 + μ*I_特殊文件)
```
- I_特殊文件 = 1当文件满足：主题归属不明确、时效要求较高或涉及资金分配

#### 步骤4：划分复核等级（高/中/低）
- 高：Score ≥ Q_70%（70%分位点）
- 中：Q_30% ≤ Score < Q_70%
- 低：Score < Q_30%

#### 步骤5：人工复核多目标优化模型

**决策变量**：
```python
x_ij = 1, 文件i分配给人员j复核; 0, 否则
y_i = 1, 文件i被人工复核; 0, 自动归档
```

**三个目标**：
- Z1 = max Σt_ij * x_ij（最大完工时间/时效性）
- Z2 = ΣΣc_ij * t_ij * x_ij（人工复核成本）
- Z3 = Σ(1-y_i) * RC_penalty（未复核错分惩罚）

**综合目标函数**：
```python
min F = α1*Z1* + α2*Z2* + α3*Z3*, α1+α2+α3=1
```

**基础约束**：
- 每个文件最多分配给一名复核人员
- 高等级文件必须复核：y_i = 1, ∀i ∈ H

**三种资源约束场景**：
| 场景 | 约束条件 |
|------|----------|
| 时间受限 | Σt_ij * x_ij ≤ T_max, ∀j |
| 预算受限 | ΣΣt_ij * x_ij ≤ B_max |
| 专家人力受限 | ΣΣt_ij * x_ij ≤ L_max, j ∈ expert |

#### 步骤6：求解方法
采用遗传算法近似求解：
- 染色体：长度为N的整数序列，g_i ∈ {0,1,...,M}
- g_i = 0：文件i不进入人工复核
- g_i = j：文件i分配给人员j
- 适应度函数：Fitness = 1 / (F + λ*P(x))

### 三种场景比较思路
| 场景 | 优先策略 |
|------|----------|
| 时间受限 | 优先处理高紧急度、高错分风险文件，舍弃低风险文件复核 |
| 预算受限 | 优先保留高等级复核，选择"风险降低收益高、复核成本低"的样本 |
| 专家人力受限 | 专家资源优先分配给特殊文件，普通文件由一般人员处理或自动归档 |
